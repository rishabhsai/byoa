# byoa

## product

BYOA lets developers run each user's Codex app-server in isolated cloud infrastructure and connect it to any product surface without shipping one shared model key.

The demo proves the smallest real integration: a user authorizes ChatGPT, gets an isolated cloud runner, and can use text or image generation from a simple app UI.

## users

- developers evaluating or integrating BYOA
- their end users connecting a personal Codex account to an app

## voice

lowercase, short, technical, and direct. explain what runs where. avoid marketing filler, inflated claims, and generic AI copy.

## visual direction

minimal, dark, dense, and utilitarian. favor clear type, thin borders, restrained yellow accents, and visible system state. avoid glossy SaaS styling, gradients, glass effects, excessive cards, oversized headings, and ornamental motion.

## interaction principles

- keyboard actions should match familiar developer tools
- streaming output should stay visible without pulling users away from history they chose to read
- connection and execution state should always be legible
- destructive or account-level actions should be explicit
- target WCAG 2.2 AA for keyboard, focus, contrast, and screen-reader behavior
